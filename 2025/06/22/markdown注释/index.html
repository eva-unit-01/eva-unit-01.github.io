Markdown实现注释有以下三种方式。为了让网页源码中不暴露注释内容，最推荐使用引用式定义实现注释。 ## html风格注释

有些markdown编辑器支持html语法，因此html语法的注释也同样支持。但是，如果markdown转换为网页之后，这些注释内容仍旧可见，因此存在泄露风险。

<div>风格注释

HTML 中的 <div> 是 "division"（中文意为“分区”或“部分”）的缩写，用作通用容器容纳其他html元素。通过设定style='display: none设定元素的行内样式属性来隐藏。但是，转换为html后虽然不可见，但是在网页源码中仍旧可见。

引用式定义实现注释

在标准的markdown语法中，插入链接常常使用行内链接，格式为[文本](url "标题")(标题是渲染成html之后，鼠标在元素上悬浮时出现的提示文字)。

其实，还有另外一种插入链接的方法称为引用式定义，这种写法把链接的「标签」和「地址」放在文档别处统一定义，正文里只写标签引用，看起来更清爽，也方便统一管理。

回到最初使用引用式定义实现注释的写法。通过把引用的url留空，markdown解析器识别到这是一个引用定义，却发现没有在文档中引用它，那么在渲染时会忽略所有没有被引用的定义。只要不引用标签，渲染成html的时候就不会在网页源码中可见。
