Flutter简述

Flutter是有Google开发的跨平台程序框架，使用dart语言进行开发。相较于其他的GUI框架，Flutter具有跨平台能力，只需要一套代码即可生成不同平台的应用程序，包括Windows，MacOS，Linux，Android，iOS以及Web（Web不支持热重载）。

Flutter环境配置

目前Flutter有较好中文文档 以及Google Codelab教程 ，可以用于自学和参考。 Flutter SDK依赖Ruby以及cocoapods包，本文将以MacOS为例进行环境配置。

Ruby安装

MacOS自带Ruby，但是只有2.x版本，而Flutter需要较新的Ruby版本。通常使用homebrew安装新版本ruby。安装完后会有一些path的设置提示:

按照提示在.zshrc中设置好homebrew安装的ruby path即可正常使用。

Ruby的官方包管理器gem所使用的源是国外镜像，出于未知原因，即便开了代理速度也很慢。可以切换为中科大镜像: 切换gem源

Cocoapods安装

如果按照Flutter教程所述使用sudo gem install cocoapods进行安装然后添加Path，则会找不到pod命令。这是因为Ruby使用homebrew安装的，两者Path并不一致。可以使用homebrew安装cocoapods：brew install cocoapods

XCode配置

按照Flutter教程中操作即可。如果不开发iOS程序，那么flutter doctor -v进行环境检测时，XCode项会提示缺少simulator，这并不影响开发桌面程序。如果需要开发iOS程序，则需要打开XCode，在左上角Windows栏选择Devices and Simulators，选择Simulators，选择+添加模拟器。

完成以上步骤之后即可编写运行第一个示例Flutter应用了。
